react和typescript项目开发规范

发表于 2019-01-28 | 分类于 react

前言

这篇文章只是针对我目前所在团队编码规范的一点建议，不具备普适性。

首先是必须遵循的几点

  • 优先设置prettier风格配置
  • 一切与prettier不可自定义 format 的配置不兼容的规则都放弃
  • 可选的几套 rules （tslint，tslint-react，tslint-microsoft-contrib，tslint-config-airbnb，tslint-config-standard，tslint-eslint-rules）
  • 暂定使用tslint，tslint-react为基准，根据团队习惯对规则进行覆盖修改
